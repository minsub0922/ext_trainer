# 정보 추출(Information Extraction) 라벨링 가이드라인

## 개요

본 가이드는 한국어 텍스트에서 정보 추출 작업을 수행하는 주석 작성자를 위한 지침입니다.

이 프로젝트에서는 다음 세 가지 정보 추출 작업을 지원합니다:
- **KV 추출 (Key-Value Extraction)**: 텍스트에서 특정 필드와 그 값을 추출
- **엔티티 추출 (Named Entity Recognition)**: 텍스트에서 사람, 조직, 위치 등의 명칭 추출
- **관계 추출 (Relation Extraction)**: 엔티티 간의 의미 관계 추출

---

## 1. 키-값(KV) 추출 작업

### 1.1 작업 설명

키-값 추출은 구조화된 문서(송장, 영수증, 청구서 등)에서 중요한 정보를 추출하는 작업입니다.

**예시:**
```
원문: "주식회사 한국테크 | 사업자등록번호: 123-45-67890 | 발행일: 2024-03-15 | 공급가액: 1,500,000원"

추출 결과:
- company_name: "주식회사 한국테크"
- business_number: "123-45-67890"
- issue_date: "2024-03-15"
- amount: "1,500,000원"
```

### 1.2 추출 규칙

1. **필드 정의 준수**: 주어진 필드 리스트만 추출합니다.
2. **정확한 텍스트 복사**: 원문에서 정확히 나타난 텍스트만 추출합니다.
3. **공백 포함**: 필드 값에 공백이 있으면 그대로 포함합니다.
4. **특수문자 처리**: 통화 기호, 구두점 등을 포함하여 복사합니다.
5. **빈 값 처리**: 문서에 해당 정보가 없으면 `null` 또는 비어있음으로 표시합니다.

### 1.3 KV 추출 예제

**예제 1: 상품 정보**
```
원문: "상품명: 프리미엄 무선 이어폰 | SKU: PW-2024-001 | 가격: 299,000원 | 재고: 45개"

필드:
- product_name: "프리미엄 무선 이어폰"
- sku: "PW-2024-001"
- price: "299,000원"
- stock: "45개"
```

**예제 2: 주문 정보**
```
원문: "주문번호: ORD-2024-567890 | 주문일시: 2024-03-14 14:30:45 | 배송지: 서울시 강남구 | 배송료: 무료"

필드:
- order_id: "ORD-2024-567890"
- order_datetime: "2024-03-14 14:30:45"
- delivery_address: "서울시 강남구"
- shipping_fee: "무료"
```

## 2. 엔티티 추출 작업

### 2.1 작업 설명

엔티티 추출은 텍스트에서 특정 유형의 명칭(사람, 조직, 위치, 날짜 등)을 찾아 표시하는 작업입니다.

### 2.2 엔티티 유형 정의

| 유형 | 설명 | 예시 |
|------|------|------|
| PERSON | 사람의 이름 | 김철수, 이순신, 스티브 잡스 |
| ORG | 조직/기업의 이름 | 삼성전자, 현대자동차, 구글 |
| LOC | 지역/위치의 이름 | 서울, 미국, 강남구 |
| DATE | 날짜/시간 표현 | 2024년 3월 15일, 어제, 다음 주 월요일 |
| PRODUCT | 제품/서비스명 | iPhone, 아이폰 14 Pro, 아메리카노 |
| MONEY | 금액 표현 | 1,000원, $100, 100만 원 |

### 2.3 엔티티 추출 규칙

1. **정확한 범위**: 엔티티의 시작과 끝 위치를 정확히 표시합니다.
2. **중복 피하기**: 동일한 엔티티가 여러 번 나타나면 모두 표시합니다.
3. **부분 추출 금지**: 전체 엔티티를 추출하되, 불필요한 수식어는 제외합니다.
   - 잘못됨: "회장 김철수" → "회장" (X)
   - 올바름: "회장 김철수" → "김철수" (O)
4. **유형 정확도**: 엔티티의 가장 적절한 유형을 선택합니다.

### 2.4 엔티티 추출 예제

**예제:**
```
원문: "삼성전자의 CEO 이재용이 2024년 3월 서울에서 신제품 발표회를 열었다."

추출 결과:
- "삼성전자" → ORG
- "이재용" → PERSON
- "2024년 3월" → DATE
- "서울" → LOC
- "신제품" → PRODUCT
```

## 3. 관계 추출 작업

### 3.1 작업 설명

관계 추출은 텍스트에서 두 엔티티 간의 의미 관계를 찾아 표시하는 작업입니다.

### 3.2 관계 유형 정의

| 관계 유형 | 설명 | 예시 |
|----------|------|------|
| works_for | ~에서 일한다 | 김철수가 삼성전자에서 일한다 |
| located_in | ~에 위치한다 | 삼성의 본사는 서울에 있다 |
| founded_by | ~에 의해 설립되었다 | 애플은 스티브 잡스가 설립했다 |
| owns | 소유한다 | 일론 머스크는 테슬라를 소유한다 |
| partner_with | ~와 파트너십을 맺는다 | 삼성과 구글이 협력한다 |

### 3.3 관계 추출 규칙

1. **주어 명확화**: 관계의 주체(head)와 객체(tail)를 명확히 구분합니다.
2. **문맥 이해**: 명시적이지 않은 관계는 문맥으로부터 추론합니다.
3. **중복 관계 표시**: 여러 관계가 존재하면 모두 표시합니다.
4. **유형 정확도**: 가장 정확한 관계 유형을 선택합니다.

### 3.4 관계 추출 예제

**예제:**
```
원문: "Apple Inc.는 Steve Jobs가 Cupertino, California에서 설립했다."

추출 결과:
1. Head: "Apple Inc." (ORG) → Relation: founded_by → Tail: "Steve Jobs" (PERSON)
2. Head: "Apple Inc." (ORG) → Relation: located_in → Tail: "Cupertino, California" (LOC)
```

## 4. 일반적인 실수와 피할 점

### 실수 1: 엔티티 경계 오류
```
✗ 잘못됨: "Apple Inc" (마침표 빠짐)
✓ 올바름: "Apple Inc." (전체 이름)
```

### 실수 2: 관계 방향 혼동
```
✗ 잘못됨: "Steve Jobs" → located_in → "Apple Inc."
✓ 올바름: "Apple Inc." → located_in → "California"
```

### 실수 3: 과도한 타입화
```
✗ 잘못됨: "애플 주식" → PRODUCT (너무 범위가 넓음)
✓ 올바름: "애플" → ORG
```

### 실수 4: 암묵적 정보 추가
```
✗ 잘못됨: 문서에 없는 정보를 추론하여 추가
✓ 올바름: 텍스트에 명시적으로 나타난 정보만 추출
```
